The Architecture

Running native binaries on Android is deceptively complex. While Android uses the Linux kernel, its userspace environment is powered by Bionic libc, not the standard Glibc found on desktop Linux.

Most "Moltis on Android" guides force you to install a 1.5GB Ubuntu distribution using proot-distro. This adds massive overhead and latency.

We chose the hard way: Native Execution. No containers, no emulation, just raw performance.

The Breakthrough: Android NDK (Bionic)

Early iterations of this project relied on Static Musl binaries to bypass Android's lack of Glibc. However, this created a "Static Wall"—the binary bypassed the Android OS socket-brokering system (VpnService), entirely breaking network requests when using Tailscale or Cloudflare.

The ultimate solution required integrating directly with Android's native C Runtime: Bionic. By utilizing modern cross-compilation toolchains (cargo-ndk) and custom C-shims, we produced a binary that natively taps into Android's internal DNS resolver and network routing tables.

Private VPS Mode

Instead of relying on public tunnels or unstable local IPs, we pivoted to a Private VPS Architecture powered by Tailscale.

By treating your phone as a node in your private mesh network, you get a stable IP that works anywhere in the world, secured by your own identity. No public ports exposed, no risk of DDoS.
